🐛 ПРОБЛЕМА
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Симптомы:
  ❌ При добавлении нескольких документов подряд (например, 8)
  ❌ Некоторые документы (1-2) не добавляются
  ❌ Данные пропадают при быстром вводе

Пример:
  Добавляешь 8 документов → сохраняется только 6-7

Причина:
  ID генерировался через Date.now()
  При быстром добавлении Date.now() возвращает одинаковое значение
  Документы с одинаковым ID перезаписывают друг друга

Технические детали:
  Date.now() возвращает миллисекунды с 1970 года
  Если вызвать 2 раза подряд быстро → одинаковое значение
  Например:
    Документ 1: id = 1699876543210
    Документ 2: id = 1699876543210 (тот же!)
    → Документ 2 перезаписывает Документ 1

Создана функция generateUniqueId():
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

let lastGeneratedId = 0;
function generateUniqueId() {
    const now = Date.now();
    // Если ID совпадает с предыдущим, добавляем 1
    if (now <= lastGeneratedId) {
        lastGeneratedId = lastGeneratedId + 1;
    } else {
        lastGeneratedId = now;
    }
    return lastGeneratedId;
}

Как работает:
  1. Получаем текущее время Date.now()
  2. Сравниваем с предыдущим ID
  3. Если совпадает → добавляем +1
  4. Если новое → используем новое значение
  5. Сохраняем последний ID

Преимущества:
  ✅ Всегда уникальные ID
  ✅ Работает при быстром вводе
  ✅ Простая реализация
  ✅ Не требует дополнительных библиотек
  ✅ Совместимо со старыми данными

Почему Date.now() не подходит?
  Date.now() возвращает миллисекунды
  Современные компьютеры выполняют код очень быстро
  Между двумя вызовами может пройти < 1 миллисекунды
  Результат: одинаковые значения

Альтернативные решения:
  1. UUID (слишком сложно)
  2. Счетчик (может сбиться)
  3. Date.now() + Math.random() (не гарантирует уникальность)
  4. generateUniqueId() (наше решение) ✅

Почему наше решение лучше?
  ✅ Простое
  ✅ Надежное
  ✅ Быстрое
  ✅ Гарантирует уникальность
  ✅ Совместимо со старыми данными
