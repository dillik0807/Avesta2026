🔧 Тест исправления дублирования клиентов

📋 Описание проблемы

Проблема: При импорте данных добавлялись дублирующиеся клиенты, потому что система неправильно проверяла существующих клиентов.

Решение: Исправлена логика проверки дубликатов с учетом двух форматов клиентов (строки и объекты с именем и телефоном).

🧪 Тестовые данные

Существующие клиенты:

1. "Иван Петров" (старый формат - строка)
2. {name: "Мария Сидорова", phone: "+992901234567"} (новый формат)
3. "ООО Стройком"

Импортируемые клиенты:

1. "Иван Петров" ❌ (дубликат)
2. {name: "Мария Сидорова", phone: "+992987654321"} ❌ (дубликат по имени)
3. "Анна Козлова" ✅ (новый)
4. {name: "Петр Иванов", phone: "+992555123456"} ✅ (новый)

🔧 Исправленная логика

// ❌ СТАРАЯ ЛОГИКА (неправильная)
if (!appData.clients.includes(client)) {
    // Добавляет дубликаты для объектов
}

// ✅ НОВАЯ ЛОГИКА (правильная)
const clientName = typeof client === 'string' ? client : client.name;
const existingClient = appData.clients.find(existingClient => {
    const existingName = typeof existingClient === 'string' 
        ? existingClient 
        : existingClient.name;
    return existingName === clientName;
});

if (!existingClient) {
    // Добавляем только если нет клиента с таким именем
}

📝 Инструкция по тестированию

  1. Создайте несколько клиентов в системе
  2. Экспортируйте данные в JSON
  3. Добавьте в экспортированный файл дублирующихся клиентов
  4. Импортируйте файл обратно
  5. Проверьте, что дубликаты не добавились
  6. Убедитесь, что новые клиенты добавились корректно

Примечание: Исправление работает для всех справочников: клиенты, товары, фирмы. Склады уже работали правильно.
