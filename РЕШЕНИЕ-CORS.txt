Ошибка в консоли браузера:
  Access to internal resource at 'file:///...manifest.json' 
  from origin 'null' has been blocked by CORS policy

Причина:
  Браузеры блокируют загрузку файлов через file:// протокол
  из-за политики безопасности CORS (Cross-Origin Resource Sharing)

Влияние:
  ⚠️ Предупреждение в консоли
  ✅ Приложение работает нормально
  ✅ Все функции доступны
  ⚠️ PWA функции недоступны при file://

✅ РЕШЕНИЕ 1: ЛОКАЛЬНЫЙ СЕРВЕР (РЕКОМЕНДУЕТСЯ)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Запустите локальный веб-сервер:

Способ A: Используйте готовый скрипт
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Windows CMD:
  start-server.bat

PowerShell:
  .\start-server.ps1

Затем откройте в браузере:
  http://localhost:8000

Способ B: Python (если установлен)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Python 3:
  python -m http.server 8000

Python 2:
  python -m SimpleHTTPServer 8000

Затем откройте:
  http://localhost:8000

Способ C: Node.js (если установлен)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Установите http-server:
  npm install -g http-server

Запустите:
  http-server -p 8000

Затем откройте:
  http://localhost:8000

Способ D: VS Code Live Server
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Установите расширение "Live Server" в VS Code
2. Откройте index.html
3. Нажмите "Go Live" в статус-баре
4. Браузер откроется автоматически

Преимущества локального сервера:
  ✅ Нет ошибок CORS
  ✅ PWA функции работают
  ✅ Service Worker работает
  ✅ Manifest.json загружается
  ✅ Полная функциональность

📝 ДОПОЛНИТЕЛЬНАЯ ИНФОРМАЦИЯ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Что такое CORS?
  Cross-Origin Resource Sharing - политика безопасности
  браузеров, которая ограничивает загрузку ресурсов
  с разных источников.

Почему file:// блокируется?
  Браузеры считают file:// протокол небезопасным для
  загрузки внешних ресурсов из соображений безопасности.

Как работает manifest.json?
  Это файл конфигурации PWA (Progressive Web App),
  который описывает приложение: название, иконки,
  цвета, режим отображения и т.д.

Нужен ли manifest.json?
  Для обычной работы приложения - НЕТ
  Для установки как PWA - ДА
  Для Android APK - НЕТ (используется capacitor.config.json)
