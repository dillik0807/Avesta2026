✅ ШАГ 1: ДОБАВИТЬ СКРИПТ В INDEX.HTML
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Откройте файл index.html и добавьте перед закрывающим тегом </body>:

    <!-- Система синхронизации в реальном времени -->
    <script src="realtime-sync.js"></script>

Место для добавления: найдите строку </body> в конце файла и добавьте
скрипт прямо перед ней.

✅ ШАГ 2: СИНХРОНИЗИРОВАТЬ ФАЙЛЫ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Запустите скрипт синхронизации:

Windows CMD:
    sync-all.bat

PowerShell:
    .\sync-all.ps1

Это скопирует realtime-sync.js во все нужные папки:
  • www/
  • avesta/www/
  • android/ (через cap sync)

✅ ШАГ 4: ПРОТЕСТИРОВАТЬ С ДВУМЯ ПОЛЬЗОВАТЕЛЯМИ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Откройте приложение в двух разных браузерах
   (или в обычном и приватном режиме)

2. Войдите в систему в обоих браузерах

3. В первом браузере:
   • Добавьте новый приход товара
   • Нажмите "Сохранить"

4. Во втором браузере:
   • Через 1-2 секунды должно появиться уведомление:
     "🔄 Данные обновлены другим пользователем"
   • Таблица автоматически обновится
   • Новый приход появится в списке

5. Если все работает - синхронизация настроена правильно! ✅

🐛 РЕШЕНИЕ ПРОБЛЕМ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Проблема: Синхронизация не работает
Решение:
  1. Проверьте интернет-соединение
  2. Откройте консоль (F12) и проверьте ошибки
  3. Убедитесь, что Firebase SDK загружен
  4. Проверьте правила доступа в Firebase Console

Проблема: Данные не обновляются у других пользователей
Решение:
  1. Проверьте индикатор статуса (должен быть зеленым)
  2. Выполните ручную синхронизацию:
     В консоли: await window.realtimeSync.manualSync()
  3. Проверьте, что оба пользователя подключены

Проблема: Частые конфликты данных
Решение:
  1. Координируйте работу пользователей
  2. Используйте разные разделы приложения
  3. Система автоматически предложит выбор при конфликте
